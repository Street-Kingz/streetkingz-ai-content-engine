# AI Assistant Rules

## Communication Style

Ben prefers:

* Short answers
* Direct answers
* Honest criticism
* Practical recommendations

Avoid:

* Corporate language
* Excessive explanations
* Over-engineering

---

## Development Philosophy

Before proposing a feature:

Ask:

1. Does it reduce workload?
2. Does it improve content output?
3. Will Ben actually use it weekly?

If not, it is likely not worth building.

---

## Project Goal

The goal is not to build software.

The goal is to create more content with less effort.

Every feature should support:

Film
→ Organise
→ Plan
→ Edit
→ Publish

Faster.

---

## Feature Prioritisation

Highest Priority:

* Content generation
* Content planning
* Content organisation
* Content gap detection

Medium Priority:

* User experience
* Reporting
* Dashboards

Low Priority:

* SaaS features
* Billing
* Multi-user support
* Enterprise functionality

---

## Technical Rules

Always verify proposed changes against the existing codebase.

Do not assume functions already exist.

Prefer updating existing workflows rather than creating parallel workflows.

Keep logic simple where possible.

---

## Future Vision

Potential future product:

AI-powered creator operating system.

Workflow:

Upload clips
→ Build content library
→ Identify gaps
→ Generate ideas
→ Generate content plans
→ Create edit packs
→ Publish content

The system should eventually support creators beyond Street Kingz without becoming dependent on Street Kingz-specific products or terminology.

NON-NEGOTIABLES

1. Do not over-engineer.
2. Build for Ben first.
3. Speed beats perfection.
4. Content output is the KPI.
5. If a feature saves less than 30 minutes per week, challenge it.
6. Prefer simple workflows over automation.
7. The user is comfortable making small manual adjustments.
8. Avoid adding setup steps.
9. The system should remain usable by one person.
10. Stop feature creep.
